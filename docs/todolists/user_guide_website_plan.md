本文档承接 [Executor 使用手册网站规划](../design/user_guide_website.md)，把网站建设拆分为可执行、可验收的任务。

## 实施原则

1. **先交付纵向切片**：优先打通“首页 → 构建 → 第一个任务 → 示例编译 → 网站部署”，不先建设大量空页面。
2. **Facade 优先**：快速开始和普通业务教程只使用 `Executor` 公开 Facade；底层接口进入高级逃生口。
3. **一个场景逐步演进**：连续教程使用机器人/数据流水线作为主线，每章只引入一个新问题。
4. **示例是受测试的产品代码**：主线完整示例进入 CMake；适合自动运行的示例进入 CTest smoke test。
5. **避免双重事实源**：构建、API、迁移和性能事实优先复用现有文档，网站负责重组学习路径。
6. **先中文后英文**：首发不因翻译阻塞，但路由和文件布局从第一天支持中英文映射。
7. **当前版本优先**：首版只维护当前开发版本；稳定发布流程形成后再评估历史版本站点。
8. **性能声明可复现**：批量、无锁、实时精度和 GPU 页面必须注明环境并链接 benchmark。
9. **失败不静默**：涉及 future、返回值、超时、背压或关闭的章节必须说明观察与处理方式。

## 风险与控制

### 内容与代码漂移

- 风险：网页片段在 API 变化后继续显示旧用法。
- 控制：完整示例纳入构建；片段从示例抽取或单独编译；发布前执行接口覆盖检查。

### 网站先于内容膨胀

- 风险：投入大量主题和组件工作，却没有可用教程。
- 控制：先完成首页和第一个任务的纵向切片；非必要自定义主题延后。

### 文档重复维护

- 风险：README、API 和网站复制相同长文后产生冲突。
- 控制：定义事实源；网站以摘要、教程和链接重组内容；自动生成或拆分前不复制完整参考。

### 教程主线过于复杂

- 风险：为了覆盖所有接口让机器人场景变成人工拼装。
- 控制：常用接口进入主线，低频接口进入专题与参考；每章只引入一个核心问题。

### 平台与硬件差异

- 风险：实时权限、CPU affinity、CUDA/OpenCL 导致用户无法复现。
- 控制：基础教程无特权、无硬件依赖；专项页面明确前置条件和降级输出。

### 多语言维护成本

- 风险：中文推进后英文长期失效。
- 控制：路由先对称、代码共用、翻译状态可见；首发明确中文优先而不是展示空英文站点。
